DoorCodes Support

Help for DoorCodes

DoorCodes keeps saved access details ready when you arrive at a place. This support page is written for the current iPhone and iPad launch scope.

Back to Home Privacy Policy

Fast checks before you contact support

Arrival alerts Use Always location plus notifications if you expect background arrivals.
Secure Reveal Protected codes require Face ID, Touch ID, or passcode authentication when enabled.
Cloud Sync Optional only. If enabled, both devices must use the same iCloud account.

Fast troubleshooting

  • If arrival alerts are not firing, make sure DoorCodes has Always location access and notification permission.
  • If Nearby works but background arrivals do not, your location setting is probably When In Use instead of Always.
  • If lock-screen alerts feel too revealing, open DoorCodes settings and change Notification Privacy.
  • If protected codes are not showing, confirm Secure Reveal is enabled and authentication is available on the device.
  • If Cloud Sync is on but data is not appearing elsewhere, verify both devices are signed into the same iCloud account and Cloud Sync is enabled inside DoorCodes.

Permissions DoorCodes may need

  • Location: used to sort nearby places and, with Always access, keep background arrival alerts reliable.
  • Notifications: used for privacy-safe arrival reminders.
  • Face ID / Touch ID / passcode authentication: used when Secure Reveal protects hidden codes.
  • iCloud: optional, only if you enable Cloud Sync.

Support Request Checklist

What to include when something goes wrong

The more concrete the report, the faster you can narrow whether the issue is permissions, sync state, or a place-specific arrival flow.

Device details

  • Your device model and iOS version
  • Whether the issue happens on iPhone, iPad, or both

App state

  • Whether Cloud Sync is enabled
  • Your DoorCodes location and notification permission states

Expected vs actual

  • Which place or arrival you expected
  • What actually happened instead
